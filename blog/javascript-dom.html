JS and its Relationship to HTML and CSS

Sticking to my previous career, I am going to use the analogy of a professional kitchen to describe JavaScript and its relationship to HTML and CSS.

In this analogy, HTML is the kitchen's layout. As in where things like your stove is, where your chopping boards are stored etc.

CSS is the plating section. It is where everything comes together and someone
has final say on presentation and what the customer, or in the case of coding, the end user, sees.

JavaScript is the pots, the pans, the knives. It is what makes everything work and function to create amazing things.

Control Flow and Loops

Control flow is referring to controlling the order in which code executes (i.e. does what it says it is going to do).

Programmers use control flow to make decisions in the code itself.

The first type of control flow is conditional statements.

The most common type of these is the if statement, and any of its derivatives (if else, if ..else if… if and nested if). One real life example of these is 


            if (I do 4 hours of programming) { 
              I will reward myself with 1 hour of playing Elden Ring
              }

Loops are most commonly used to execute code a specific amount of times.

A real life example of a loop:

	For (start at minutes  0; if minutes are greater than  25; add one minute){
              Go take a 10 minute study break
            }

Where the first section inside the normal () is declaring an iterator.

The second section is stating when this loop should end when it meets the critera.

The third section is what the loop does after completing one loop-around

The sections are seperated by ; marks

The DOM

The DOM is an object which stores all the elements of a webpage and allows you to access and interact with them.

Your web browser will structure these elements into a web-like representation when it sees you are loading an HTML document.

This is where JavaScript comes into play. You can use JS to change and manipulate content, style and structure of the webpage.

An example snippet of code:

document.body.style.background = : “black”

This will change the background colour of the webpage without accessing a CSS stylesheet.

Differences Between Arrays and Objects

Arrays are single type element (data) “lists” while objects are multi-type element (data) “labels”.

Arrays are accessed from an ordered list of items while objects are labels which store the data in an unordered fashion.

Each label has its own value of any type.

Functions

Functions are re-usable code that helps reduce a program’s complexity via its reusability.

It must be defined in one place and then ‘called’ to action some other place in the program.

They are capable of taking parameters - pieces of information which the function can manipulate
or read to use in its inner code to do all kinds of things with.

Though taking parameters is not required and functions can work without them the same way.

They also may return a piece of data, although this is not required for said function to function (lol).
