This is a post of some stories about some problems I have been stuck on the last week, and how I solved them.

As well as a breakdown of some problem-solving techniques and how comfortable I am using them.

A Story about a Time I was Blocked on A Simple Problem

What was the problem?

This week I had a lot of trouble with a simple problem. The problem was with a function which would take a set of test scores and assign them to a student.

Basically I was trying to add an object property to an already existing object property.
But the main issue I had was figuring out a way to find the correct object to add the test scores to.

What problem-solving techniques did you use?

The problem-solving technique I used was a lot, a lot, a lot of googling. Seriously every google search even remotely close to this problem is purple now.

How did you feel throughout the process?

Quite frustrated to be honest. I thought “Surely this can’t be that complicated, it’s only question 3 on the 4th kata!”
but eventually I came across the .hasOwnProperty() method in an MDN web doc.

This quickly solved my issue alongside a little help from Dave about dot versus bracket notation.

What did you learn?

I learned that I might need to work on breaking questions down and get better at explaining my problems.
I found it quite difficult to explain what I was actually struggling with here at the start,
and I think I luckily stumbled upon the .hasOwnProperty() method rather than any kind of skilled googling. Something to work on in the future!

A Story about a Time I elegantly Solved a Problem

What was the problem?

The problem I elegantly solved this week was the stretch question in the gradebook kata.
This kata involved creating a new gradebook from scratch in one function.
An empty object, an array of student names and an array of test scores were passed into a function as arguments.

What problem-solving techniques did you use?

The problem-solving technique I used was trying something out.
I did a few attempts at completing the function before I realised I was basically doing the same as question 3.
So I copy-pasted my code from question 3 into the new enterScores function and it worked! .forEach() is a powerful method

How did you feel throughout the process?

To be honest I was a bit chuffed when I copy pasted the code into the function and it worked.
Throughout the process, I think I was overthinking the problem at first but once I broke what the function was actually asking to do,
I realised I had basically solved it already.

What did you learn?

I learned that sometimes in programming, the best answer is a simple one.
Instead of overcomplicating everything with for loops and if else if ladders, you can use pre built-in methods and such to complete logic puzzles.

A Reflection on a time I was Reluctant to ask for Help

I was reluctant to ask for help with the getScore function in the gradebook kata. I think I was reluctant because I thought I could do it myself, and I can be a bit stubborn like that sometimes. But eventually I realised it would be better to just ask a question I thought was silly, rather than sitting for a few hours to complete the question by brute forcing it. Next time I try to break the question down into simpler terms and ask questions more often.
